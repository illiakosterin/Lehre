Python has become the common language of quantitative finance, data science, and AI research. This course teaches you to use it properly — not just to run code that works, but to understand why it works and how to adapt it when the problem changes. We go from the basics of the language all the way through to deploying a model that someone else can actually interact with.

Every dataset we work with, every problem we model, is drawn from finance. That is not just a pedagogical choice — it reflects how I think about data science. The methods are general, but the judgment required to apply them well is domain-specific. Knowing when a regression is the right tool, how to interpret a clustering result on market data, or what a sentiment model is actually capturing in financial text — that only comes from working with real problems in the field, not toy examples.

The course is structured so that each module builds directly on the previous one. By the time we reach deployment, students have already written the model, validated it, and understood its limitations. That sequence matters. In practice, the most dangerous moment is when someone deploys a model they do not fully understand. We try to make sure that does not happen here.
